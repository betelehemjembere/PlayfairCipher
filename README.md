# Playfair Cipher Encryption System

A modern and interactive implementation of the classical Playfair Cipher encryption algorithm using HTML, CSS, and JavaScript.

This project demonstrates how symmetric encryption techniques work by allowing users to encrypt and decrypt messages using a keyword-generated 5×5 matrix. The application includes a clean responsive interface, matrix visualization, automatic text preparation, and full Playfair Cipher functionality.


#  Project Overview

The Playfair Cipher is a manual symmetric encryption technique invented in 1854 by Charles Wheatstone. Unlike simple substitution ciphers, Playfair encrypts pairs of letters (digraphs), making frequency analysis more difficult and improving security compared to monoalphabetic ciphers.

This project was developed as part of a Computer Security / Cryptography assignment to demonstrate understanding of classical encryption algorithms and secure communication concepts.

# How Playfair Cipher Works

The Playfair Cipher encrypts text using pairs of letters instead of single letters.



## Step 1 — Generate Matrix

A 5×5 matrix is generated using a keyword.

Example keyword:
    MONARCHY


Generated matrix:

    M O N A R
    C H Y B D
    E F G I K
    L P Q S T
    U V W X Z




## Step 2 — Prepare Plaintext

Before encryption, the plaintext is processed using the following rules:

 Convert all letters to uppercase
 Replace `J` with `I`
 Split letters into pairs
 Insert `X` between repeated letters
 Add `X` if the message length is odd

Example:
HELLO
prepared text → HELXLO




## Step 3 — Encryption Rules

### Same Row Rule

If both letters are in the same row, replace each with the letter to its right.

### Same Column Rule

If both letters are in the same column, replace each with the letter below it.

### Rectangle Rule

If letters form a rectangle, replace each with the letter in the same row but opposite corner.

# Security Concept

The keyword acts as the secret key in the Playfair Cipher.

Different keywords generate different matrices, meaning the same plaintext encrypted with different keywords produces completely different ciphertexts.

Without the correct keyword, proper decryption becomes impossible.
